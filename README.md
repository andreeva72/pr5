# flower_shop — модульное CLI-приложение на Dart

Консольное приложение для управления цветочным магазином с хранением данных в SQLite.

## Предметная область

Система автоматизации цветочного магазина:

- пользователи (`User`);
- категории (`Category`);
- товары (`Product`) с привязкой к категории;
- корзина (`Cart`) с привязкой к пользователю;
- элементы корзины (`CartItem`) с привязкой к корзине и товару;
- заказы (`Order`) с привязкой к пользователю;
- элементы заказа (`OrderItem`) с привязкой к заказу и товару.

## Архитектура и структура папок

Проект разделен на модули по слоям:

```text
lib/
  flower_shop.dart         # публичный экспорт
  src/
    models/                # сущности
      user.dart
      category.dart
      product.dart
      cart.dart
      cart_item.dart
      order.dart
      order_item.dart
    repositories/          # CRUD операции
      user_rep.dart
      category_rep.dart
      product_rep.dart
      cart_rep.dart
      cart_item_rep.dart
      order_rep.dart
      order_item_rep.dart
    database/              # SQLite и создание таблиц
      database.dart
    cli/                   # меню и обработка ввода
      menu.dart
      input_helper.dart
    validators/            # валидация
      text_valid.dart
      number_valid.dart
      date_valid.dart
bin/
  main.dart                # точка входа (main)
test/
  domain_test.dart         # тесты сущностей
  data_test.dart           # тесты БД
  validation_test.dart     # тесты валидации

### Что вынесено в каждый слой и почему

- `models`: описывает данные и преобразования (`toMap`/`fromMap`), не зависит от SQL и консоли.
- `repositories`: содержат CRUD-операции для каждой сущности, работают с БД.
- `database`: отвечает за открытие БД, создание таблиц и FOREIGN KEY связи.
- `cli`: отвечает за взаимодействие с пользователем (`stdin`/`stdout`) и навигацию по меню.
- `validators`: чистые функции для проверки данных (текст, числа, даты).
- `bin`: только запуск приложения и закрытие ресурсов.

Такое разделение упрощает поддержку, тестирование и замену инфраструктуры (например, SQLite на другой способ хранения).

## Реализованная функциональность

- просмотр списка пользователей, категорий, товаров, корзины, заказов;
- добавление и удаление пользователей;
- добавление и удаление категорий;
- добавление и удаление товаров;
- добавление и удаление товаров в корзину;
- оформление заказа из корзины;
- вывод всех таблиц БД в консоль.

## Валидация

Реализованы проверки:

- обязательный текстовый ввод: значение не пустое после `trim()`;
- числовые поля для товара: цена и количество должны быть корректными числами больше 0;
- дата/время: поддерживается `DateTime.parse()` для даты заказа;
- email: должен содержать символ `@`;
- телефон: минимум 10 символов.

## Тесты

В проекте есть 3 обязательных типа тестов:

1. `toMap/fromMap` для сущности `User`;
2. подключение к базе данных;
3. валидный и невалидный email, проверка числа больше 0.
